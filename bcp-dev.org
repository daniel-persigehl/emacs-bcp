* Package Overview :world:
The BCP — Biblical Commentary and Prayer framework is an Emacs environment
for liturgical prayer and Bible study. It fetches scripture from Oremus
Bible Browser or local files, renders Daily Office liturgies, and supports
annotated Bible study with org-roam cross-references.

** Design Principles :constraint:
- Rubric texts are NEVER altered — only shown/hidden
- Prayer/canticle texts are stored constants, never generated
- Variant forms are separate named constants
- Buffer is text-mode, read-only, overlays for all styling
- Spacing normalised post-render by bcp-liturgy--normalise-spacing

* Domain Architecture :world:
:PROPERTIES:
:STATUS: SETTLED
:END:

The codebase is divided into four domains. The boundary rule is:
if a module requires liturgical machinery to make sense, it belongs
under =bcp-liturgy-=. If it is genuinely reusable outside a liturgical
context, it belongs under =bcp-=.

** =bcp-= Shared Framework
Infrastructure with no domain semantics. Serves all other domains.
- Pure computus (Easter, dominical letter, moveable feasts)
- Western rank taxonomy
- Oremus fetch/parse layer
- Buffer rendering primitives
- User preferences

** =bcp-liturgy-= Liturgical Office (generic)
The generic Office machinery shared across all rites.
Not tied to any specific prayer book edition.
- Generic ordo walker and renderer
- Canticle registry (texts only meaningful in an ordo context)
- Propers protocol (the standard plist shape all rites return)
- Rite implementations live under their own sub-prefix:
  - =bcp-1662-=           Book of Common Prayer 1662
  - =bcp-breviary-=       Breviarium Romanum 1962
  - =bcp-anglican-breviary-=  Anglican Breviary (1955)
  - =bcp-lobia-=          Little Office of the BVM
  - =bcp-divine-worship-= Divine Worship: Daily Office

** =bcp-reader-= Scripture Reader
Simple, fast text display for study reading.
Passage navigation, translation management, local file loading.
No ordo, no appointed texts — just "show me this passage."

** =bcp-notebook-= Annotation and Cross-Reference
org-roam nodes, marginal note capture, backlink browser,
contextual views, spanning psalm editor, export.
Can in principle attach notes to any displayed text —
Office lessons, collects, reader passages alike.

* bcp-fetcher.el :science:
:PROPERTIES:
:STATUS: SETTLED
:END:

Core fetch/parse layer. No dependencies on bible-commentary or bcp-1662.
Must be required first. Both packages use its symbols directly.

** Public API
- bcp-fetcher-fetch(passage callback &optional translation)
  Async. Returns propertized string. Properties PRESERVED.
  Do NOT clean — would strip wrap-prefix/display properties.
- bcp-fetcher-fetch-for-commentary(passage translation callback)
  Async. Applies clean-text. For commentary buffer only.
- bcp-fetcher-dom-to-text(node book &optional start-psalm)
  DOM → formatted string with text properties.
- bcp-fetcher-clean-text(text)
  Cleans Windows-1252 chars. Strips text properties.
- bcp-fetcher-fetch-oremus(passage load-fn &optional translation)
  Async fetch for commentary buffer, calls load-fn(text label).
- bcp-fetcher-fetch-oremus-context(ref translation load-fn banner-fn)
  Context-window fetch for omitted verses with footnotes.
- bcp-fetcher-oremus-process(passage translation load-fn)
  Core fetch-and-process used by the above.
- bcp-fetcher-active-translation(ref)
  Returns correct translation for REF per priority rules.
- bcp-fetcher-textual-status(ref), bcp-fetcher-textual-header-annotation
  Textual-critical status for omitted/disputed verses.
  Cleans Windows-1252 chars. Strips text properties.

** Critical Constraint :constraint:
bcp-fetcher-clean-text and replace-match strip ALL text properties.
For Office buffer, DOM results returned DIRECTLY without cleaning.
In-buffer char cleaning in insert-text-block handles survivors.

** Defcustoms (canonical home — all fetch config lives here)
- bible-commentary-translation ("KJVA") — default translation
- bible-commentary-psalm-translation ("Coverdale") — auto-used for Psalms
- bible-commentary-oremus-base-url, bible-commentary-oremus-context-url
- bible-commentary-oremus-version-codes, bible-commentary-omitted-verse-context-window
- bible-commentary-critical-translation-names, bible-commentary-vulgate-translation-names
- bible-commentary-orthodox-translation ("NRSV") — for LXX-only books

** Rank taxonomy :constraint:
Full pre-1955 Western taxonomy in bcp-rank-order (19 levels,
highest → lowest): sunday-1cl, double-1cl, octave-1cl, vigil-1cl,
sunday-2cl, double-2cl, octave-2cl, vigil-2cl, greater-double,
sunday-common, double, semidouble, octave-3cl, vigil-common,
feria-privileged, feria-major, simple, feria-minor, commemoration.

Aliases in bcp-rank-aliases map BCP (principal/greater/lesser),
1962 classes, and OF names to canonical symbols.

Utilities: bcp-rank-value, bcp-rank<, bcp-rank<=, bcp-rank-higher,
bcp-rank-canonical, bcp-rank-description.

** Spacing Model :constraint:
- Each insert fn emits content + \n only
- Walk loop inserts \n after each step with output
- normalise-spacing collapses 3+ newlines → 2
- insert-rubric: self-checks, adds leading \n if needed, emits \n\n after
- insert-text-block: NO leading blank — heading flows directly to text

** insert-text-block Post-Processing
1. Strip display text properties (verse marker alignment)
2. Clean Windows-1252 chars in-buffer
3. Set wrap-prefix per line (verse: measured; canticle: "   ")

** insert-canticle-text
Prepends "   " before insert-text-block. Aligns first verse
(col 0 in stored text) with subsequent verses (start \n   ).

** Versicles use ℣ (U+2123) and ℟ (U+211F).
