* Package Overview :world:
The BCP — Biblical Commentary and Prayer framework is an Emacs environment
for liturgical prayer and Bible study. It fetches scripture from configurable
backends (default: Oremus Bible Browser) or local files, renders Daily Office
liturgies, and supports annotated Bible study with org-roam cross-references.

** Design Principles :constraint:
- Rubric texts are NEVER altered — only shown/hidden
- Prayer/canticle texts are stored constants, never generated
- Variant forms are separate named constants
- Buffer is text-mode, read-only, overlays for all styling
- Spacing normalised post-render by bcp-liturgy--normalise-spacing

* Domain Architecture :world:
:PROPERTIES:
:STATUS: SETTLED
:END:

The codebase is divided into four domains. The boundary rule is:
if a module requires liturgical machinery to make sense, it belongs
under =bcp-liturgy-=. If it is genuinely reusable outside a liturgical
context, it belongs under =bcp-=.

** =bcp-= Shared Framework
Infrastructure with no domain semantics. Serves all other domains.
- Pure computus (Easter, dominical letter, moveable feasts)
- Western rank taxonomy
- API-agnostic fetch layer (backends register via bcp-fetcher-register-backend)
- Buffer rendering primitives
- User preferences

** =bcp-liturgy-= Liturgical Office (generic)
The generic Office machinery shared across all rites.
Not tied to any specific prayer book edition.
- Generic ordo walker and renderer
- Canticle registry (texts only meaningful in an ordo context)
- Propers protocol (the standard plist shape all rites return)
- Rite implementations live under their own sub-prefix:
  - =bcp-1662-=              Book of Common Prayer 1662
  - =bcp-1928-=              American BCP 1928
  - =bcp-roman-=             Roman Office (DA 1911 is test case)
  - =bcp-roman-lobvm-=       Little Office of the BVM
  - =bcp-roman-1962-=        Breviarium Romanum 1962 (future)
  - =bcp-roman-1568-=        Breviarium Romanum 1568 (future)
  - =bcp-roman-sarum-=       Use of Sarum (future)
  - =bcp-roman-monastic-=    Monastic Breviary (future)
  - =bcp-anglican-breviary-= Anglican Breviary 1955 (future)
  - =bcp-divine-worship-=    Divine Worship: Daily Office (future)

** =bcp-reader-= Scripture Reader
Simple, fast text display for study reading.
Passage navigation, translation management, local file loading.
No ordo, no appointed texts — just "show me this passage."

** =bcp-notebook-= Annotation and Cross-Reference
org-roam nodes, marginal note capture, backlink browser,
contextual views, spanning psalm editor, export.
Can in principle attach notes to any displayed text —
Office lessons, collects, reader passages alike.

* bcp-fetcher.el :science:
:PROPERTIES:
:STATUS: SETTLED
:END:

API-agnostic fetch dispatch layer. No dependencies on bible-commentary or bcp-1662.
Must be required first. Auto-loads bcp-fetcher-oremus at the bottom.
Backends register themselves via bcp-fetcher-register-backend; active backend
is selected by the defcustom bcp-fetcher-backend (default: 'oremus).

** Backend Protocol
A backend is registered as:
  (bcp-fetcher-register-backend 'NAME
    :name "Display Name"
    :fetch-fn #'my-backend--fetch     ; (fn passage translation callback)
    :translations '("KJVA" "NRSV" …)) ; for completing-read

The :fetch-fn must call (callback text) where text is a propertized string
(verse numbers formatted for visual-line-mode) or nil on failure.

** Public API
- bcp-fetcher-fetch(passage callback &optional translation)
  Async. Dispatches to active backend. PRESERVES text properties.
  Do NOT clean result — would strip wrap-prefix/display properties.
- bcp-fetcher-fetch-passage(passage load-fn &optional translation)
  Async. Calls load-fn(text label). Used by bcp-reader / bcp-notebook.
- bcp-fetcher-fetch-passage-context(ref translation load-fn banner-fn)
  Context-window fetch for omitted/disputed verses. Banner-fn called
  immediately with status string; load-fn called with text + label.
- bcp-fetcher-clean-text(text)
  Cleans Windows-1252 C1 chars and nbsp. Text properties preserved.
- bcp-fetcher-register-backend(name &rest plist)
  Register a fetch backend. See Backend Protocol above.
- bcp-fetcher-available-translations()
  Returns translation list from the active backend (for completing-read).
- bcp-fetcher-active-translation(ref)
  Returns correct translation for REF per priority rules.
- bcp-fetcher-textual-status(ref), bcp-fetcher-textual-header-annotation
  Textual-critical status for omitted/disputed verses.

** Critical Constraint :constraint:
bcp-fetcher-clean-text and replace-match strip ALL text properties.
For Office buffer, DOM results returned DIRECTLY without cleaning.
In-buffer char cleaning in insert-text-block handles survivors.

** Cache Constraint :constraint:
Passage cache keys are (passage . translation). Changing psalm translation
or backend via the transient menu must clear the cache (bcp-fetcher-clear-cache)
to prevent stale results from the fallback chain being served.

** Translation-Aware Routing :constraint:
bcp-fetcher-fetch checks if the requested translation matches any registered
backend's :translations list (via bcp-fetcher--backend-for-translation).
If a match is found and it differs from the primary backend, that backend is
tried first in the fallback chain. This means setting psalm-translation to
"Vulgate" automatically routes psalm fetches to the vulgate backend without
requiring a change to bcp-fetcher-backend.

** Vulgate Backend
Registered as 'vulgate. Translations: ("Vulgate" "Latin").
Serves from bcp-liturgy-psalter-vulgate.txt (Vulgate numbering, 150 psalms).
Applies BCP-to-Vulgate psalm number mapping via bcp-fetcher--bcp-to-vulgate:
| BCP 1-8 | Vulgate 1-8 (same) |
| BCP 9 | Vulgate 9 (full, covers BCP 9+10) |
| BCP 10 | nil (subsumed by Vulgate 9) |
| BCP 11-113 | Vulgate N-1 |
| BCP 114 | Vulgate 113 vv.1-8 |
| BCP 115 | Vulgate 113 vv.9-end |
| BCP 116 | Vulgate 114 + 115 concatenated |
| BCP 117-146 | Vulgate N-1 |
| BCP 147 | Vulgate 146 + 147 concatenated |
| BCP 148-150 | Vulgate 148-150 (same) |
Reuses bcp-fetcher--coverdale-render-psalm for propertized text rendering.

** Defcustoms (canonical home — translation config lives here)
- bcp-fetcher-backend ('coverdale) — active backend symbol
- bible-commentary-translation ("KJVA") — default translation
- bible-commentary-psalm-translation ("Coverdale") — auto-used for Psalms; "Vulgate"/"Latin" for Latin
- bible-commentary-omitted-verse-context-window (2) — context verses for omissions
- bible-commentary-critical-translation-names — triggers textual-status warnings
- bible-commentary-vulgate-translation-names — triggers Vg psalm numbering
- bible-commentary-orthodox-translation ("NRSV") — for LXX-only books

- bible-commentary-oremus-base-url, bible-commentary-oremus-context-url
- bible-commentary-oremus-version-codes

** bcp-fetcher-oremus.el
Oremus backend. Loaded automatically by bcp-fetcher.el.
Owns: URL templates, version code mapping, HTML DOM parsing,
bcp-fetcher-oremus--dom-to-text, extract-bibletext, strip-fallback.
Registers as 'oremus backend on load.

** Spacing Model :constraint:
- Each insert fn emits content + \n only
- Walk loop inserts \n after each step with output

** insert-text-block Post-Processing
1. Strip display text properties (verse marker alignment)
2. Clean Windows-1252 chars in-buffer
3. Set wrap-prefix per line (verse: measured; canticle: "   ")

** insert-canticle-text
Prepends "   " before insert-text-block. Aligns first verse
(col 0 in stored text) with subsequent verses (start \n   ).

* Roman Office (bcp-roman-) :science:
:PROPERTIES:
:STATUS: IN-DEVELOPMENT
:END:

Roman Breviary framework, DA 1911 rubrics. Source: Divinum Officium project.
Pre-extracted data (not runtime DO parsing). All eight LOBVM hours complete.

** Architecture
Parallel to Anglican side:
| bcp-roman-ordo.el | Step type definitions + ordines for all 8 hours |
| bcp-roman-render.el | Step dispatch, office walker, language threading |
| bcp-roman-lobvm.el | LOBVM data (Latin + English) + entry points |
| bcp-common-roman.el | Shared Roman fixed texts (Latin + English) |
| bcp-roman-hymnal.el | Hymn registry: multiple translators per hymn |

** Roman ordo step types
:rubric, :text, :versicles (reused from Anglican), plus Roman-specific:
:incipit, :antiphon, :psalm, :psalm-group, :hymn, :capitulum, :canticle,
:confession, :absolution, :pre-oratio, :collect, :conclusio,
:marian-antiphon, :commemoratio, :silent-prayers, :invitatory,
:lesson, :responsory, :te-deum, :lesson-absolutio, :lesson-benedictio,
:nocturn-versicle.

** LOBVM Hours
All eight canonical hours implemented:
| Hour | Entry point | Transient key |
| Matins | =bcp-roman-lobvm-matins= | m |
| Lauds | =bcp-roman-lobvm-lauds= | l |
| Prime | =bcp-roman-lobvm-prime= | p |
| Terce | =bcp-roman-lobvm-terce= | t |
| Sext | =bcp-roman-lobvm-sext= | s |
| None | =bcp-roman-lobvm-none= | n |
| Vespers | =bcp-roman-lobvm-vespers= | v |
| Compline | =bcp-roman-lobvm-compline= | c |

=bcp-roman-lobvm= auto-selects the appropriate hour by time of day.
All hours accessible from the LOBVM transient submenu (R from main menu).

Fixed texts embedded (psalms, hymns, capitula, collects).
Seasonal variation via data resolver:
| per-annum | Sub tuum praesidium, Alleluia, Salve Regina |
| advent | Sub tuum praesidium, Alleluia, Alma Redemptoris Mater |
| christmastide | Sub tuum praesidium, Alleluia, Alma Redemptoris (different V./R. + collect) |
| lent | Sub tuum praesidium, Laus tibi Domine, Ave Regina caelorum |
| eastertide | Regina caeli (psalm antiphon), Alleluia, Regina caeli (final) |

** Multilingual Support :constraint:
Controlled by =bcp-roman-office-language= defcustom (default: ='latin=).
Toggleable from transient menu (L key in LOBVM submenu).

Three strategies by text type:
1. *Office prose* (antiphons, collects, versicles): Marquess of Bute 1908
   translation as English fallback, stored as parallel =-en= constants.
   Resolver tries English first, falls back to Latin.
2. *Hymns*: Separate hymnal file (=bcp-roman-hymnal.el=), keyed by Latin
   incipit, supporting multiple named English translations per hymn.
   User selects preferred translator via =bcp-roman-hymnal-preferred-translator=
   defcustom (default: ='britt=). Fallback chain: =bcp-roman-hymnal-fallback-order=.
   Available translators: Britt, Caswall, Neale, Primer.
3. *Scripture* (capitula, lessons): Store only the citation reference (:ref slot).
   English text resolved at render time via =bcp-fetcher-fetch= using the
   user's configured Bible translation. Latin Vulgate text stays embedded.

All headings, rubrics, and structural text localised:
Psalmus/Psalm, Hymnus/Hymn, Capitulum/Chapter, Lectio/Lesson,
secreto/said silently, Absolutio/Absolution, Benedictio/Blessing, etc.

Canticles (Nunc dimittis, Magnificat, Benedictus, Benedicite) fetch
English text from the canticle registry via =:canticle-key= in data plists.

Marian antiphons have a parallel English alist (=bcp-roman-marian-antiphons-en=)
with Bute 1908 verse translations.

** Penitential Forms Registry :constraint:
Cross-tradition registry in bcp-common-prayers.el:
=bcp-penitential-forms= alist, registered via =bcp-penitential-register=.
Registered forms: roman, anglican-1662, anglican-1662-lay (pardon and peace),
anglican-1928.
Defcustoms =bcp-roman-confession-form= and =bcp-roman-absolution-form=
control which form the Roman renderer uses. Works in reverse too.

** Marian Antiphons
Four seasonal final antiphons in bcp-common-roman.el (Latin + English):
| advent | Alma Redemptoris Mater | Angelus Domini / Et concepit |
| christmastide | Alma Redemptoris Mater | Post partum / Dei Genetrix |
| lent | Ave Regina caelorum | Dignare me laudare / Da mihi virtutem |
| eastertide | Regina caeli | Gaude et laetare / Quia surrexit |
| per-annum | Salve Regina | Ora pro nobis / Ut digni efficiamur |
Each has antiphon + V./R. + Oremus + collect.
English translations (=bcp-roman-marian-antiphons-en=) from Bute 1908.

* Rejected Approaches
:PROPERTIES:
:STATUS: SETTLED
:END:

** REJECTED org-mode Buffer
Fontification overwrites overlays; marker hiding caused unpredictable
spacing. → Switched to text-mode.

** REJECTED Drawer-based Fixed Texts
org-mode folds drawers. → Plain text insertion.

** REJECTED Post-DOM Character Cleaning
replace-match strips ALL text properties including wrap-prefix.
→ Clean HTML before DOM parsing; in-buffer for survivors.

** REJECTED include-chapter-headings Parameter in dom-to-text
Recursive calls didn't propagate parameter, broke verse numbering.
→ Deferred as known cosmetic issue.

** REJECTED dom-texts for Office Lessons
Loses all verse number formatting. → Must use bcp-fetcher-dom-to-text.
